# 微纳加工与光学测试实验记录系统

适用对象：

- Dirac-vortex cavity；
- Disclination cavity；
- EBL 微纳加工；
- ICP/RIE 刻蚀；
- SEM 表征；
- SOI 光学测试。

核心目标：

```text
任何 SEM 图
→ 反查样品
→ 反查 GDS
→ 反查 Python 脚本
→ 反查 Dose / PEC / 工艺参数
→ 判断结果
→ 决定下一步
```

避免以后出现这些无法追溯的文件名：

- `final.gds`
- `final_v2.gds`
- `SEM_new_final.tif`

## 1. 核心原则

实验记录必须围绕 **Sample（样品）**，不是围绕单独的 SEM、ICP 或 GDS 文件。

正确逻辑：

```text
Sample
├── GDS
├── EBL
├── ICP
├── SEM
└── Optical
```

原因：一个样品会经历完整工艺链。只记录单个环节，后期会断掉追溯关系。

## 2. 样品命名规则

推荐格式：

```text
B批次_日期_基底_结构来源_工艺阶段_样品编号_结果状态
```

示例：

```text
B15_20260522_Si_DiscFig2_EBLICP_S1_OK
```

字段解释：

| 字段 | 含义 |
|---|---|
| `B15` | Batch15 |
| `20260522` | 日期 |
| `Si` | 基底类型 |
| `DiscFig2` | Disclination cavity, Nature Photonics 2024 Fig.2 |
| `EBLICP` | 当前完成 EBL + ICP 主流程 |
| `S1` | 样品编号 |
| `OK` | 当前结果可用 |

常见基底：

| 字段 | 含义 |
|---|---|
| `Si` | 硅片 |
| `SOI` | Silicon-on-insulator，用于光学测试样品 |
| `Quartz` | 石英 |
| `InP` | 磷化铟 |

常见结构：

| 字段 | 含义 |
|---|---|
| `DiscFig2` | Disclination cavity, Fig.2 |
| `DVFig3` | Dirac-vortex cavity, Fig.3 |
| `DVDisc` | Dirac-vortex + Disclination 组合结构 |

结果字段：

| 字段 | 含义 |
|---|---|
| `OK` | 初步成功 |
| `FAIL` | 失败 |
| `TEST` | 测试样品 |
| `OPT` | 光学测试 |
| `Optical` | 面向光学测试的样品 |

## 3. GDS 与 Python 追溯

每个样品必须记录：

| 项目 | 必填内容 |
|---|---|
| GDS 文件 | 实际用于曝光的 `.gds` 文件 |
| GDS 简写 | 放进 SEM 文件名里的参数短名 |
| Python 脚本 | 生成 GDS 的 `.py` 文件 |
| 生成命令 | 以后能重新生成同一个 GDS |
| 参数 JSON | 如果有，记录参数文件 |

当前 B15 示例：

| 项目 | 内容 |
|---|---|
| 样品 | `B15_20260522_Si_DiscFig2_EBLICP_S1_OK` |
| GDS | `disclination_a559_r0p2_n3p4_extended_100um.gds` |
| GDS 简写 | `a559r0p2n3p4` |
| 实际确认脚本 | `20-Projects/02_Comsol_samulation/scripts_20260521_1/comsol_mph_to_gds.py` |
| 旧记录脚本 | `20-Projects/02_Comsol_samulation/scripts/dxf_to_gds.py` |

> [!warning]
> 当前 `scripts/dxf_to_gds.py` 已不在项目中。B15 追溯时优先使用实际确认脚本 `scripts_20260521_1/comsol_mph_to_gds.py`。

## 4. 结构参数记录

必须记录结构参数，不要只写“Fig.2”。

| 参数 | 含义 | 示例 |
|---|---|---|
| `a` | 周期 | `559 nm` |
| `r` | 孔半径或孔径比例 | `0.2*a` |
| `n_substrate` | 折射率参数 | `3.4` |
| `extended` | 扩展版图尺寸 | `100 um` |
| 腔尺寸 | 结构实际尺寸 | 待补充 |
| 阵列大小 | 版图阵列大小 | 待补充 |

> [!note]
> 文件名里的 `n3p4` 表示 `n_substrate=3.4`。这里的 `p` 是小数点，不是分数线。

## 5. 区域编号 Region

一个样品上通常有多个版图、多个结构或多个 Dose，所以必须建立区域编号。

推荐格式：

```text
R01, R02, R03...
```

含义：

| 区域编号 | 含义 |
|---|---|
| `R01` | 第一个版图区域 |
| `R02` | 第二个版图区域 |
| `R03` | 第三个版图区域 |

当前 B15 区域示例：

| 区域编号 | 对应 GDS | Dose | 基底 | 结果 |
|---|---|---|---|---|
| `R01` | `disclination_a559_r0p2_n3p4_extended_100um.gds` | 90% | Si | OK |

目标：SEM 文件名中的 `R01` 可以直接反查到 GDS、Dose、基底和结果。

## 6. SEM 文件命名规则

错误命名：

- `SEM_01.tif`
- `final_SEM.tif`
- `P1_1.tif`

这些名字的问题：

- 不知道结构；
- 不知道 Dose；
- 不知道区域；
- 不知道基底；
- 不知道是否成功。

推荐格式：

```text
B15_20260522_Si_DiscFig2_a559r0p2n3p4_D90_R01_center_30k_OK_001.tif
```

字段解释：

| 字段 | 含义 |
|---|---|
| `B15` | Batch15 |
| `20260522` | 日期 |
| `Si` | 基底 |
| `DiscFig2` | 结构来源 |
| `a559r0p2n3p4` | GDS 关键参数 |
| `D90` | Dose 90% |
| `R01` | 区域编号 |
| `center` | SEM 拍摄位置 |
| `30k` | SEM 倍率 |
| `OK` | 当前结果 |
| `001` | 第几张图 |

SEM 位置字段：

| 字段 | 含义 |
|---|---|
| `center` | 结构中心 |
| `edge` | 结构边缘 |
| `corner` | 版图角落 |
| `overview` | 低倍全局 |
| `defect` | 缺陷位置 |
| `unknown` | 暂时不知道 |

当前 B15 原始 SEM 示例：

| 原 SEM 文件名 | 建议新文件名 | 说明 |
|---|---|---|
| `BATCH15_20260522_EBL_ICP_RIE_S1_P1_1.tif` | `B15_20260522_Si_DiscFig2_a559r0p2n3p4_D90_R01_unknown_unknown_OK_001.tif` | 位置和倍率暂时未知，先不要正式重命名。 |

> [!warning]
> 不要直接重命名原始 SEM 图。先生成重命名建议表，人工确认区域、位置、倍率后，再复制到 `重命名图/`。

## 7. 工艺记录逻辑

原则：固定参数可以引用标准工艺，变化参数必须单独记录。

必须重点记录：

### 7.1 涂胶

示例：

| 项目 | 示例 |
|---|---|
| 胶 | AR-P6200.09 |
| 第一步转速 | 500 rpm, 5 s |
| 第二步转速 | 2000 rpm, 60 s |
| 前烘 | 150 ℃, 60 s |

### 7.2 EBL

| 项目 | 示例 |
|---|---|
| Dose | 90% |
| PEC | ON/OFF |
| Beam current | 2 nA |
| 显影 | 待补充 |

术语说明：

- `Dose`：电子束曝光剂量。
- `PEC`：Proximity Effect Correction，邻近效应修正，用来补偿电子散射造成的局部过曝或欠曝。

### 7.3 ICP

如果完全使用标准工艺，可以写：

```text
标准 ICP
```

如果改动了气体、功率、压力、刻蚀时间，必须记录。

### 7.4 RIE 去胶

这是最容易忘记的参数之一。

示例：

| 项目 | 示例 |
|---|---|
| 去胶方式 | RIE |
| 气体 | O2 |
| 时间 | 60 s |

或者：

| 项目 | 示例 |
|---|---|
| 去胶方式 | 丙酮 |
| 时间 | 10 min |
| 是否超声 | 是/否 |

## 8. Dose Matrix 记录

Dose Matrix 是“剂量矩阵”，用来比较不同 GDS 和不同曝光剂量。

推荐记录方式：

| 区域 | GDS | Dose | PEC | 预期观察 | SEM 结果 |
|---|---|---|---|---|---|
| A1 | GDS01 | 25% | OFF | 中心是否孔连 | 待补充 |
| A2 | GDS01 | 30% | OFF | 中心是否孔连 | 待补充 |
| A3 | GDS01 | 35% | OFF | 中心是否孔连 | 待补充 |
| B1 | GDS02 | 25% | OFF | 边缘是否正常 | 待补充 |

目标：

```text
SEM
→ 反查区域
→ 反查 GDS + Dose + PEC
```

## 9. PEC 记录

对于 JEOL JBX-6300FS，必须记录：

- PEC 是 ON 还是 OFF；
- Dose Matrix；
- 是否出现中心孔连或局部过曝。

Disclination 中心区域尤其容易出现：

- 孔连；
- 局部过曝；
- 图形合并；
- 中心区域形貌失真。

## 10. 超净间记录系统

现实限制：

- 超净间只能带手机；
- 有 A4 纸；
- 有纸质小本；
- 纸本不能带出再带回。

推荐系统：

```text
纸质实时记录
+
手机拍照备份
+
Obsidian 当天整理
```

## 11. 纸质本记录什么

纸质本只记录现场会变化、容易忘、必须当场记的信息。

推荐记录：

| 类型 | 示例 |
|---|---|
| 实时参数 | `Dose=90`, `PEC OFF` |
| 临时改动 | `显影少 5 s`, `前烘多 10 s`, `ICP 功率变化` |
| 现场观察 | `胶边缘不均匀`, `中心过曝` |
| 草图 | `R01/R02` 在样品上的位置 |
| 时间点 | `14:32 开始显影`, `15:01 ICP 结束` |

这些信息绝不能靠记忆。

## 12. 手机 / Obsidian 记录什么

手机和 Obsidian 负责长期数据库信息。

推荐记录：

| 类型 | 示例 |
|---|---|
| 样品身份证 | `B15_20260522_Si_DiscFig2_EBLICP_S1_OK` |
| GDS | `disclination_a559_r0p2_n3p4_extended_100um.gds` |
| Python | `comsol_mph_to_gds.py` |
| 工艺结论 | `中心孔连`, `PEC OFF` |
| 下一步 | `PEC ON`, `降低 Dose`, `SOI 光学测试` |

## 13. 推荐工作流

### Step 1：超净间内

纸质本只记录变化。

示例：

```text
Dose=90
PEC OFF
显影 55 s
R01 中心过曝
```

### Step 2：出超净间

立刻用手机拍纸质记录。

### Step 3：当天晚上

整理进 Obsidian，形成正式实验记录。

> [!warning]
> 不要“以后再整理”。48 小时后很多细节会忘。

## 17. 最终目标

建立完整样品数据库，实现：

```text
任何 SEM
→ 样品
→ 基底
→ GDS
→ Python
→ Dose
→ PEC
→ 工艺
→ 结果
→ 下一步
```

这样 Dirac-vortex、Disclination、SOI、Optical 测试都不会混乱。
