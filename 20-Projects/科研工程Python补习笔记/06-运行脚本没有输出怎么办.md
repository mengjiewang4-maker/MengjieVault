# 06-运行脚本没有输出怎么办

运行脚本没有输出，不一定是失败。

可能情况：

1. 脚本确实没有 `print`
2. 输出写到了文件
3. 图被 `plt.show()` 阻塞或没有弹出
4. 代码只定义函数，没有调用
5. 分支条件没有进入
6. 输出路径不是你以为的位置

## 先找输出语句

```bash
rg -n "print\\(|write_gds|savefig|plt.show|to_csv|np.savetxt|open\\(|write\\(" script.py
```

如果完全没有这些关键词，脚本可能只是模块。

## 看有没有真正执行

如果文件只有：

```python
def main():
    ...
```

但没有：

```python
if __name__ == "__main__":
    main()
```

那么直接运行可能什么都不做。

## 看输出文件在哪里

搜索路径变量：

```bash
rg -n "output|out_dir|save_path|gds_path|Path\\(|mkdir" script.py
```

很多 AI 生成脚本会写：

```python
out_dir = Path("outputs")
```

这表示输出在“当前运行目录”的 `outputs/`，不是脚本所在目录。

## 运行前后对比文件

运行前：

```bash
find . -type f | sort > before.txt
```

运行脚本：

```bash
python script.py
```

运行后：

```bash
find . -type f | sort > after.txt
diff before.txt after.txt
```

这样可以知道有没有新文件生成。

## 建议给脚本加的最小输出

如果你自己写脚本，至少加：

```python
print("Start")
print(f"Output: {output_path}")
print("Done")
```

GDS 脚本建议输出：

```python
print(f"Wrote {gds_path}")
print(f"Number of holes: {len(holes)}")
print(f"Dose: {dose_percent}%")
```

## 对旧脚本的判断

没有输出的旧脚本，可以先归类为：

| 现象 | 判断 |
|---|---|
| 只有函数 | 模块 |
| 有顶层计算但无 print | 可运行探索脚本 |
| 有输出函数但找不到文件 | 输出路径需确认 |
| 有 `plt.show()` | 可能只弹图，不写文件 |
