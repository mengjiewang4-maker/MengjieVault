# 09-陌生项目三分钟检查清单

## 0. 先不要运行

先看项目结构：

```bash
find . -maxdepth 3 -type f | sort
```

## 1. 找 Python 文件

```bash
find . -name "*.py" | sort
```

记录：

- 有多少个 py？
- 是否分文件夹？
- 文件名是否有 `main`、`run`、`generate`、`export`？

## 2. 找标准入口

```bash
rg -n "if __name__|def main|argparse" .
```

如果没有，不代表不能运行，只是说明项目不规范。

## 3. 找 import 关系

```bash
rg -n "^import |^from .* import" .
```

标记：

- 外部库
- 项目内部模块
- 当前目录找不到的模块

## 4. 找输出

```bash
rg -n "write_gds|savefig|plt.show|to_csv|np.savetxt|write\\(|open\\(" .
```

看脚本最终会不会生成：

- `.gds`
- `.png`
- `.csv`
- `.txt`
- `.lsf`

## 5. 判断文件身份

| 文件 | 判断 |
|---|---|
| 有 main 或顶层执行代码 | 主程序/可运行脚本 |
| 主要是 def/class | 模块 |
| 被很多文件 import | 模块 |
| 输出 GDS/CSV/PNG | 可能是入口 |

## 6. 找 README 和参数记录

```bash
find . -name "*.md" -o -name "*.txt" | sort
```

优先读：

- README
- 参数表
- 运行记录
- 文件对应关系
- 实验记录

## 7. 运行前确认

运行前回答：

1. 我从哪个目录运行？
2. 入口文件是哪一个？
3. 需要哪些第三方库？
4. 输出会写到哪里？
5. 会不会覆盖已有文件？

## 8. 最小安全运行方式

先用只读检查或 dry-run：

```bash
python script.py --help
python script.py --dry-run
```

如果没有这些选项，先读代码里的输出路径，不要盲跑。
