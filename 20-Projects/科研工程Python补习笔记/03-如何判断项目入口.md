# 03-如何判断项目入口

项目入口就是“应该从哪个 `.py` 文件开始运行或阅读”。

科研项目里入口不一定叫 `main.py`，尤其是 AI/Codex 生成的代码，可能会有很多看起来都能运行的脚本。

## 入口文件常见名字

优先关注：

```text
main.py
run.py
demo.py
generate_*.py
export_*.py
pipeline.py
*_workflow.py
*_demo.py
*_preview.py
```

GDS 项目里常见：

```text
generate_gds.py
*_gds_export.py
*_layout.py
*_dose_matrix.py
```

仿真项目里常见：

```text
export_comsol.py
export_lumerical.py
run_simulation.py
build_structure.py
```

## 用关键词找入口

```bash
rg -n "if __name__|def main|argparse|write_gds|savefig|plt.show|to_csv|np.savetxt|open\\(" .
```

解释：

- `if __name__`：标准入口
- `def main`：主函数
- `argparse`：命令行参数，常见于入口
- `write_gds`：GDS 输出
- `savefig`/`plt.show`：画图输出
- `to_csv`/`np.savetxt`：数据输出

## 判断优先级

| 现象 | 入口可能性 |
|---|---|
| 有 `if __name__ == "__main__"` | 很高 |
| 有 `def main()` | 很高 |
| 顶层直接写 GDS/CSV/PNG | 高 |
| 文件名含 `generate`、`export`、`run`、`workflow` | 高 |
| 只有函数定义，没有顶层执行 | 低，通常是模块 |
| 被很多文件 import | 低，通常是模块 |

## 没有标准入口怎么办

很多科研脚本没有：

```python
if __name__ == "__main__":
```

但仍然能运行，因为顶层代码会直接执行。

例如：

```python
a = 0.56
holes = generate_holes(a)
write_gds(holes, "test.gds")
```

这类文件是“可运行脚本”，但不是规范主程序。

## 推荐入口不等于最佳代码

推荐入口只是“最适合开始阅读/运行”的文件，不代表它代码质量最好。

一个入口可能：

- 能跑，但结构混乱
- 参数写死
- 没有日志
- 没有 README
- 输出路径不清楚

阅读时先用入口理解流程，再回头整理模块。
