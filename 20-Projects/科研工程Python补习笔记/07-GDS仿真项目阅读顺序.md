# 07-GDS/仿真项目阅读顺序

GDS/仿真项目一般不是普通软件工程，它的核心是“参数 -> 几何 -> 导出 -> 实验/仿真记录”。

## 推荐阅读顺序

1. README 或实验记录
2. 输出文件名
3. 参数表
4. 主生成脚本
5. 几何函数
6. 导出函数
7. 后处理/画图
8. 历史版本

## 先找输出

GDS 项目先搜：

```bash
rg -n "write_gds|GdsLibrary|Cell|Round|Polygon|gds" .
```

仿真项目先搜：

```bash
rg -n "lumerical|comsol|fdtd|mode|export|csv|lsf|mat" .
```

## GDS 脚本重点看什么

| 内容 | 为什么重要 |
|---|---|
| 单位 | um、nm 混乱会导致版图尺寸错误 |
| 周期 `a` | 结构尺度核心参数 |
| 孔径/线宽 | 直接影响 EBL 和 SEM |
| 阵列尺寸 | 决定总结构大小 |
| 中心区域处理 | disclination cavity 最敏感 |
| cell 名称 | 导入版图软件时方便识别 |
| 输出路径 | 防止覆盖旧 GDS |
| 文件名规则 | 对应样品编号和实验记录 |

## 仿真脚本重点看什么

| 内容 | 为什么重要 |
|---|---|
| 输入坐标 | 是否来自正确 GDS/CSV |
| 材料参数 | Si、SiO2、空气孔等 |
| 边界条件 | PML、周期边界会影响结果 |
| 网格尺寸 | 影响精度和时间 |
| 波长范围 | 是否覆盖目标模式 |
| 输出数据 | 谱线、场分布、Q 值、模式体积 |

## 历史版本不要直接跑

历史脚本常见问题：

- 参数写死
- 输出路径写死
- 会覆盖旧 GDS
- import 已经失效
- 没有版本号
- 和当前实验记录不对应

先读 README 和参数表，再决定是否运行。

## 建议给每个 GDS 项目建立 3 张表

1. Python -> GDS 对应表
2. GDS -> 参数对应表
3. GDS -> Dose/PEC/区域对应表

这样后续 SEM 回看时不会断链。
