# 文献管理说明

本页定义当前 vault 的长期文献管理约定，目标是让 `PDF -> 来源页 -> 概念/综述` 这条链条稳定可维护。

## 推荐目录结构

- `学习资料/论文/待整理/`：新进 PDF 的默认落点
- `学习资料/论文/已整理/`：已经建立正式 `来源/` 页的 PDF
- `学习资料/书籍/`：教材、专著、习题与长篇参考书
- `学习资料/剪藏/已整理/`：已清洗并沉淀的网页来源
- `学习资料/剪藏/待清洗/`：暂不可信或还没清洗的网页
- `来源/`：正式来源笔记
- `综述/`：主题综述与总览
- `系统/索引/文献整理索引.md`：原始资料与正式来源页映射

## 命名规则

- PDF 默认保留原始文件名，减少外部检索与人工识别成本
- `来源/` 页面优先使用文献原题名
- 综述页统一使用 `主题总览`、`主题应用版图`、`路线比较`、`指标框架`
- 索引或规范页统一使用 `说明`、`索引`、`清单`

## 来源页 Frontmatter 模板

```yaml
---
title: 文献标题
type: source
status: to-read
created: 2026-04-20
updated: 2026-04-20
tags:
  - literature
  - source
  - paper
  - optics
sources:
  - 学习资料/论文/待整理/文献标题.pdf
aliases:
  - 常用简称
---
```

书籍可将 `paper` 改为 `book`；剪藏来源可将 `paper` 改为 `clipping`。

## 标签体系

- 基础标签：`literature`、`source`、`review`
- 类型标签：`paper`、`book`、`clipping`
- 学科标签：`optics`、`quantum`、`embedded`
- 主题标签：`vortex-light`、`topological-photonics`、`photonic-device`

## 状态体系

- `to-read`：已入库但还没开始处理
- `reading`：正在读，已有基础信息但未形成完整摘要
- `reviewed`：已完成摘要级整理，可用于概念页和综述页
- `cited-in-review`：已被综述页稳定引用，可作为主题骨架来源

## 最小工作流

1. 新 PDF 进入 `学习资料/论文/待整理/`
2. 建立对应 `来源/` 页并写入 frontmatter
3. 完成摘要级整理后，将 PDF 移到 `学习资料/论文/已整理/`
4. 把来源页挂回 `综述/`、`概念/` 或 `比较/`
5. 在 [[文献整理索引]] 登记映射关系
