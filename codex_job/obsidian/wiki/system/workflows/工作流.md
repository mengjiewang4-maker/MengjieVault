# 工作流

## Ingest

适用场景：用户指定 `学习资料/` 中的某个来源，要求处理、摘要、整理、吸收。

标准步骤：
1. 先读 [[Home_Index|index]] 和 [[ResearchVaultshare/wiki/log]]
2. 读取指定来源
3. 在根目录合适子目录中创建或更新来源摘要页
4. 更新相关概念页、综述页或比较页
5. 回写 [[Home_Index|index]]
6. 在 [[ResearchVaultshare/wiki/log]] 追加 ingest 记录

产出要求：
- 说明来源讲了什么
- 说明哪些结论值得进入长期知识库
- 说明与现有页面的关系
- 标出不确定点、争议点、适用前提

## Query

适用场景：用户问一个问题，希望基于现有知识库作答。

标准步骤：
1. 先看 [[Home_Index|index]]
2. 读相关 wiki 页面
3. 必要时回源到 `学习资料/`
4. 形成答案
5. 若答案具有长期价值，则沉淀为新页面
6. 在 [[ResearchVaultshare/wiki/log]] 记录 query 或 refactor

## Lint

适用场景：用户要求检查知识库健康度。

检查清单：
- 是否存在未登记到 [[Home_Index|index]] 的页面
- 是否存在缺少 `sources` 的知识页
- 是否存在没有任何内部链接关系的孤立页
- 是否存在应拆分或合并的重复页面
- 是否存在新来源已推翻旧结论但页面未更新
- 是否有反复出现、但尚未独立成页的概念

## 文件组织建议

- 先少量高质量页面，再逐步扩展
- 对长期主题建立综述页，避免每次都从零找资料
- 对单个来源建立来源摘要页，避免以后重复读长 PDF
