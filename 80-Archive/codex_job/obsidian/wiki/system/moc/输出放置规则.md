# 输出放置规则

以后新增输出，按下面规则放。

## 1. 图片结果

放到：`science/results/`

适用：
- Python 绘图输出
- 分析结果图
- 需要单独查看的 PNG、JPG、SVG

建议命名：
- `主题-日期.png`
- `实验名-v1.png`

## 2. Markdown 成品

放到各项目自己的输出目录：

- `science/obsidian/输出/`
- `mengjie-viki-new/obsidian/outputs/summaries/`
- `mengjie-viki-new/obsidian/outputs/reviews/`

适用：
- 实验报告
- 阶段总结
- review
- synthesis

## 3. 工程生成物

原则上放到项目目录下单独的输出目录，例如：

- `工作区/Project_Replication_Vortex/output/`

适用：
- `.gds`
- 中间导出文件
- 批处理生成的交付物

说明：
- 当前该项目的 `.gds` 已统一放到 `工作区/Project_Replication_Vortex/output/`
- 后续如果要继续整理，继续保持源码与 `output/` 分开
- 迁移前需要同步检查脚本和 README

## 4. 不要混放

下面几类内容尽量分开：

- 源码不要和输出混放
- 原始资料不要和成品混放
- 临时草稿不要和最终交付物混放
