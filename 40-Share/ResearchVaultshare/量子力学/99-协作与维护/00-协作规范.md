# 量子力学学习仓库协作说明

## 仓库定位

本仓库是一个：

- 面向长期学习的量子力学知识库

- 基于 Obsidian 的可双链知识网络

- 可结合 Codex / AI 协助整理与学习

- 支持多人协作维护


目标不是“简单记笔记”，而是逐步构建：

- 可检索

- 可推导

- 可复习

- 可扩展

- 可协作


的系统化知识体系。

---

# Git 分支规范

本仓库使用：

```text
main
dev
feature/*
```

结构。

---

## main 分支

用途：

- 稳定知识库

- 已整理完成内容

- 可直接学习与分享


要求：

- 不直接开发

- 不直接提交临时内容

- 保持结构稳定


---

## dev 分支

用途：

- 日常整理与合并

- 所有功能先进入 dev

- 测试无问题后再进入 main


要求：

- 每次开始工作前先 pull

- 不长期堆积未整理内容


---

## feature 分支

用途：

- 某个专题知识整理

- 某项功能开发

- 某类笔记重构


命名规范：

```text
feature/专题名称
```

例如：

```text
feature/angular-momentum
feature/schrodinger-equation
feature/python-visualization
feature/zotero-import
feature/latex-cleanup
```

---

# 推荐协作流程

## 1. 更新仓库

```bash
git checkout dev
git pull
```

---

## 2. 创建专题分支

```bash
git checkout -b feature/angular-momentum
```

---

## 3. 整理内容

仅修改：

- 当前专题

- 当前任务相关文件


避免大规模修改整个仓库。

---

## 4. 提交修改

```bash
git add .
git commit -m "整理角动量基础结构"
git push
```

---

## 5. Pull Request

```text
feature/*
→ dev
```

review 后：

```text
dev
→ main
```

# Obsidian 文件规范

## 文件命名

统一使用：

```text
00-索引.md
01-xxx.md
02-xxx.md
```

不要混用：

- 最终版.md

- new.md

- test2.md


---

# 图片管理

统一放入：

```text
figures/
assets/
attachments/
```

图片命名：

```text
fig_wavepacket.png
fig_harmonic_oscillator.png
```

不要使用：

```text
微信图片.png
截图1.png
```

---

# Markdown 规范

## 图片引用

统一：

```markdown
![[figures/wavepacket.png]]
```

---

## 双链

统一：

```markdown
[[波函数]]
[[薛定谔方程]]
```

避免：

- 同一概念多个名字

- 中英文混乱


---

# Frontmatter 规范

统一：

```yaml
---
tags:
  - quantum
concepts:
  - 波函数
status: permanent
---
```

---

# Codex / AI 使用规范

允许：

- 整理专题

- 拆分知识结构

- 自动生成索引

- Latex 清理

- Python 示例生成


不建议：

- 直接重构整个仓库

- 批量重命名全部文件

- 自动覆盖所有 frontmatter


避免：

- 双链断裂

- Dataview 失效

- 文件结构混乱


---

# 不建议直接协作修改的内容

以下文件尽量避免多人同时修改：

```text
00-索引.md
README.md
AGENTS.md
```

建议：

- 指定负责人维护

- 其他人通过 PR 修改
