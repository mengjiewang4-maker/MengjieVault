# 船用通导系统为什么不是设备堆砌

## 目标（预计 5-10 分钟）

读完后，你应该能用一个闭环模型解释船用通导系统：它不是把雷达、ECDIS、AIS、GPS、罗经、计程仪、测深仪、VDR、BNWAS 摆上船，而是让船舶在不确定海况中持续回答四个问题：

- 我在哪里？
- 周围有什么？
- 我该怎么安全航行？
- 发生异常后，谁来报警、记录和复盘？

## 核心问题

为什么一艘船需要这么多通导设备？为什么不能只装一个大屏幕，或者只靠 GPS 和雷达？

答案是：航行安全不是单点信息问题，而是一个“多源感知、交叉校验、辅助决策、记录追责”的闭环问题。

单个设备只能回答一小类问题。例如：

- DGPS/北斗回答“本船大概在哪里、当前时间是什么”。
- 罗经回答“船头朝哪里”。
- 计程仪回答“船相对水或地的速度是多少”。
- 测深仪回答“船底离海底还有多深”。
- 雷达回答“周围目标相对我在哪里”。
- AIS 回答“周围安装 AIS 的船自报了什么身份和动态信息”。
- ECDIS 把海图、航线、本船位置、雷达或 AIS 目标组织到可决策的画面里。
- VDR 记录关键数据，事故后可复盘。
- BNWAS 监控驾驶员值班状态，防止无人值守。

如果只看设备清单，很容易学成“背型号”。如果看闭环，就能开始判断系统是否完整、接口是否合理、冗余是否足够。

## 第一性原理/核心模型

船用通导系统的第一性原理是：船舶在海上运动时，必须持续降低四类不确定性。

第一类是不确定自己的状态：位置、航向、航速、姿态、水深。  
第二类是不确定外部环境：其他船、岸线、障碍物、浅水区、天气。  
第三类是不确定行动后果：当前航线是否安全，是否会碰撞，是否偏航。  
第四类是不确定责任链条：谁值班、什么时候报警、事故后数据是否可追溯。

因此可以把系统压缩成一个闭环：

```text
传感器采集 -> 数据分发 -> 显示/融合 -> 人或自动系统决策 -> 报警/记录 -> 复盘改进
```

对应到船用通导设备：

| 闭环环节 | 典型设备 | 解决的问题 |
|---|---|---|
| 传感器采集 | DGPS、北斗、电罗经、磁罗经、计程仪、测深仪、雷达、AIS | 获取位置、航向、航速、水深、目标 |
| 数据分发 | NMEA 分配器、RS-422、以太网、接线盒 | 把数据送到需要它的设备 |
| 显示/融合 | ECDIS、Conning、雷达显示器 | 把分散数据变成可判断的态势 |
| 决策执行 | 自动操舵仪、驾驶员、DP/推进系统接口 | 保持航向、跟踪航线、避碰 |
| 报警/记录 | BNWAS、VDR、浅水报警、系统报警 | 防失职、防遗漏、可追溯 |

看 H2623 技术协议时，最重要的不是先记住“某型号有几套”，而是问：这台设备在闭环里处在哪一环？输入来自谁？输出给谁？断掉以后影响什么能力？

## 关键概念与例子

概念 1：通导设备的价值由“数据位置”决定。

一台 DGPS 不只是一个定位盒子。它输出的位置、对地速度、航向或时间，可能送到雷达、电罗经、AIS、VDR、测深仪、计程仪、ECDIS、自动舵、DP 系统等。也就是说，它的价值不只在显示本船经纬度，而在于成为很多设备的共同数据源。

如果 DGPS 数据质量差，受影响的不是一个屏幕，而是后续显示、记录、报警、航迹控制和系统联动。

概念 2：系统集成的难点常在接口，而不是单机功能。

例如 AIS 的基本功能是接收和发送船舶静态、动态、航线及安全信息。但在系统里，它还要接收电罗经、DGPS、计程仪数据，并把 AIS 目标输出到雷达、ECDIS、VDR。若接口格式、波特率、句子类型、线缆屏蔽、接地或信号分配有问题，单机合格也不等于系统可用。

一个简单例子：

```text
DGPS -> AIS -> ECDIS
电罗经 -> AIS -> ECDIS
计程仪 -> AIS -> VDR
雷达 -> VDR
测深仪 -> ECDIS/VDR
```

这里任何一个箭头都是集成检查点。箭头不是“画图装饰”，而是数据责任。

## 常见坑

1. 把“设备清单”误认为“系统架构”

设备清单只能说明买了什么，不能说明数据怎么流、故障怎么隔离、冗余怎么切换、试验怎么验收。真正的架构要能回答“输入、输出、依赖、失效后果”。

2. 只看主设备，不看分配器、接线盒、电源单元和天线

通导系统的大量问题发生在边界处：天线位置、电缆长度、NMEA 分配器、RS-422 接线、220VAC/24VDC 电源、接地和电磁兼容。技术协议里反复出现电源单元、信号分配器、天线电缆，不是细枝末节。

3. 只看单机证书，不看系统级试验

CCS 产品证书说明设备本身满足某类要求，但装船后的接口、报警、记录、显示、联动还需要通过系泊试验、航行试验和报验程序闭环。

4. 把 AIS 当成雷达替代品

AIS 依赖对方主动发送信息，雷达依赖目标反射。AIS 更像“协作目标信息”，雷达更像“非协作目标感知”。两者互补，不能简单互相替代。
